# 关键链路

## 项目创建链路

项目通过以下命令创建：

```powershell
android create empty-activity --name="AndroidProject" --output=.\AndroidProject
Rename-Item .\AndroidProject AgenticLedger
```

创建结果：

- 项目目录：`D:\workspace\selfspace\Androids\AgenticLedger`
- 应用模块：`app`
- 默认包名：`io.github.cming0420.agenticledger`

## 开发链路

1. 在项目根目录查看工程结构：

```powershell
android describe --project_dir=.
```

2. 修改 `app/src/main/java/com/example/androidproject/` 下的 Kotlin 代码。

3. 修改界面资源或配置时，检查 `app/src/main/res/` 与 `AndroidManifest.xml`。

4. 涉及文档、构建或运行方式变化时，同步更新相关文档。

## 功能链路

文本消费记账：

```text
消费文本输入 -> ExpenseAgent -> record_expense 结构化消费账单 -> SQLite -> Compose 列表刷新
```

文本收入记账：

```text
收入文本输入 -> ExpenseAgent -> record_income 结构化收入账单 -> SQLite -> Compose 列表刷新
```

Agent 配置：

```text
设置页填写界面标题/base_url/api_key/model_name -> 保存到应用私有 SQLite -> 下次解析时生效
```

导入配置保护：

```text
覆盖导入包含设置的 ZIP -> 设置标记为导入来源 -> base_url/api_key/model_name 遮罩且禁用 -> 重置清空并解除锁定
```

日期范围查询：

```text
账单页打开日期区间选择器 -> 选择日期或快捷范围 -> ViewModel 本地闭区间筛选 -> 显示账单与净额
```

Dashboard 统计与展示：

```text
账单/Dashboard 共享日期范围 -> ViewModel 基于筛选后账单生成 DashboardStats -> Compose 渲染 KPI 卡片 -> Vico 渲染折线图/饼图/双柱状图
```

账单编辑删除：

```text
账单页长按账单项 -> 编辑弹窗 -> 保存金额/事项/渠道/分类/日期时间到 SQLite 或删除记录 -> Compose 列表刷新
```

数据导出：

```text
设置页选择账单/设置 -> 账单可选日期范围 -> 生成 JSON payload -> AES-GCM 加密 -> 写入 ZIP -> 系统保存文件
```

覆盖导入：

```text
设置页选择 ZIP 备份 -> 弹出覆盖提示 -> 解压并解密 payload -> 按文件内容覆盖 SQLite 中的账单或设置
```

底部导航与主题：

```text
底部图标 NavigationBar 切换记账/账单/Dashboard/设置 -> AnimatedContent 页面过渡 -> 设置页保存跟随系统/亮色/暗色
```

界面交互：

```text
主要按钮/账单卡片按压 -> Compose UI 层动画反馈 -> ViewModel 仅处理业务状态
```

## 构建链路

在项目根目录执行：

```powershell
.\gradlew.bat assembleDebug
```

调试包通常输出在：

```text
app/build/outputs/apk/debug/
```

## 测试链路

运行本地单元测试：

```powershell
.\gradlew.bat test
```

运行设备或模拟器测试前，先确认设备状态和目标 APK，再使用 Android CLI 或 Gradle 相关命令执行。

## 运行链路

推荐流程：

1. 使用 Gradle 构建 APK。
2. 使用 Android CLI 查询或确认构建产物。
3. 使用 `android run` 将应用部署到设备或模拟器。

已连接真实设备 `eef7206f` 时可执行：

```powershell
.\gradlew.bat assembleDebug
android run --device=eef7206f --apks=app\build\outputs\apk\debug\app-debug.apk
```

## 发布链路（GitHub Actions）

### 自动触发

推送 `v*` 标签时自动构建并创建 Release：

```powershell
git tag v1.0.0
git push origin v1.0.0
```

### 手动触发

在 GitHub 仓库的 Actions 页面手动运行 "Build & Release" 工作流。

### 构建产物

- **APK**: `app/build/outputs/apk/release/*.apk`
- **AAB**: `app/build/outputs/bundle/release/*.aab`

### 签名配置

Release 构建使用签名，需在 GitHub 仓库 Settings → Secrets and variables → actions 中配置以下 Secrets：

| Secret 名称 | 说明 |
|-------------|------|
| `KEYSTORE_BASE64` | keystore 文件的 Base64 编码 |
| `KEYSTORE_PASSWORD` | keystore 密码 |
| `KEY_ALIAS` | 密钥别名 |
| `KEY_PASSWORD` | 密钥密码 |

生成 `KEYSTORE_BASE64`：

```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("release.jks"))
```

本地构建签名 APK（需设置环境变量）：

```powershell
$env:KEYSTORE_PASSWORD="<your-password>"
$env:KEY_ALIAS="<your-alias>"
$env:KEY_PASSWORD="<your-password>"
.\gradlew.bat assembleRelease
```

### 工作流步骤

1. 检出代码
2. 设置 JDK 17
3. 配置 Gradle（含缓存）
4. 解码 keystore 文件
5. 运行 Lint 检查
6. 构建签名 Release APK
7. 构建签名 Release AAB
8. 创建 GitHub Release 并上传产物
