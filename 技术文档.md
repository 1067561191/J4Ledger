# 技术文档

## 技术栈

- 项目类型：Android 文本收支记账应用
- 创建工具：Android CLI
- 模板：`empty-activity`
- 语言：Kotlin
- UI：Jetpack Compose
- 图表：Vico 3.1.0 Compose / Material 3
- 构建系统：Gradle Kotlin DSL
- 应用模块：`app`
- 当前包名：`com.example.androidproject`
- 最低系统版本：Android 12 / API 31
- Agent：OpenAI-compatible Chat Completions + `tool_calls`
- 本地存储：应用私有 SQLite
- 备份格式：ZIP + AES-GCM 对称加密 payload
- 显示模式：跟随系统、亮色、暗色

## 当前模块结构

- `MainActivity.kt`：应用入口 Activity。
- `Navigation.kt`、`NavigationKeys.kt`：导航相关代码。
- `data/`：账单模型、SQLite 仓储、备份导入导出和 Agent 解析。
- `theme/`：Compose 主题、颜色和字体定义。
- `ui/main/`：文本记账、底部导航、页面过渡动画、日期范围账单、Dashboard 统计图、账单编辑删除、Agent 设置、主题模式和主界面 ViewModel。

## 核心流程

1. 用户在输入框填写自然语言消费或收入。
2. `MainScreenViewModel` 调用 `ExpenseAgent`。
3. 如果 `base_url`、`api_key`、`model_name` 已配置，则请求 OpenAI-compatible `/chat/completions`，要求模型按入口调用 `record_expense` 或 `record_income` 工具返回结构化字段。
4. 如果未配置或远程调用失败，则显示友好提示，不写入账单。
5. `SQLiteLedgerRepository` 将账单保存到本地 SQLite，并通过 `StateFlow` 更新界面。
6. 账单页通过日期区间选择控件做闭区间筛选并显示净额，支持今天、本周、本月、本季度、年度快捷范围。
7. Dashboard 复用同一个日期筛选状态，ViewModel 对筛选后的账单构建 `DashboardStats`，并用 Vico 展示每日收支趋势、分类占比、渠道收支对比和收入支出占比。
8. 账单项支持长按打开编辑弹窗，可修改金额、事项、渠道、分类/收入类型和日期时间，也可删除。
9. 账单金额按类型展示：消费使用 `-` 前缀，收入使用 `+` 前缀并以深黄色显示。

## 备份流程

- 导出：`SQLiteLedgerRepository.exportBackup` 从 SQLite 读取配置和账单，按选项生成 JSON payload，使用 AES-GCM 加密后写入 ZIP 的 `payload.enc`，同时写入 `manifest.json`。
- 导入：`SQLiteLedgerRepository.importBackup` 读取 ZIP，找到 `payload.enc` 后自动解密，再在事务内覆盖文件中包含的设置或账单数据。
- UI：设置页通过 Android 系统文件选择器保存或打开 `.zip` 备份文件，导入前弹出覆盖确认。

## 导航与主题

- 主界面使用底部 `NavigationBar` 作为 `记账`、`账单`、`Dashboard`、`设置` 的导航入口。
- 页面切换使用横向滑入和淡入淡出的 `AnimatedContent` 过渡。
- `AgentSettings.themeMode` 存储用户选择的显示模式，SQLite v2 通过 `theme_mode` 字段保存。
- SQLite v3 为账单增加 `entry_type` 字段，为设置增加 `imported_from_backup` 字段，用于导入配置的敏感字段遮罩和禁用输入。
- 设置通过备份导入后，`base_url`、`api_key`、`model_name` 在设置页显示为 `********` 且不可编辑；重置配置会清空界面标题和 Agent 配置，并解除导入锁定。

## Dashboard 统计

- 数据来源：只基于当前 `LedgerFilter` 筛选后的 SQLite 账单，不新增数据库字段。
- 每日收支趋势：日期范围内按天补齐，消费和收入分别进入两条折线。
- 分类占比：消费和收入分别按 `category` 汇总，各自显示饼图。
- 渠道收支对比：按 `channel` 汇总，超过 8 个渠道时保留前 7 个并将其余合并为“其他渠道”。
- 收入支出占比：使用收入总额与消费总额的绝对金额，不使用净额符号。

## 数据格式

应用仍处于未发布阶段，不再兼容第一版 SharedPreferences 存储，也不会在启动时做旧数据转换。当前有效数据格式以 SQLite 表结构和加密 ZIP 备份 schema 为准。

## 构建与识别

使用 Android CLI 识别工程：

```powershell
android describe --project_dir=.
```

使用 Gradle Wrapper 构建：

```powershell
.\gradlew.bat assembleDebug
```

## 后续扩展约定

- 新增业务界面时，优先放在 `app/src/main/java/com/example/androidproject/ui/` 下按功能分包。
- 新增数据源、仓储或模型时，优先放在 `data/` 或按业务建立清晰包结构。
- 修改导航入口时，同步检查 `Navigation.kt` 和 `NavigationKeys.kt`。
- 引入新依赖时，同步检查 `gradle/libs.versions.toml`、模块 `build.gradle.kts` 和本文档。
- 如需将包名从模板默认值改为正式命名空间，应同步调整源码目录、测试目录、`namespace`、`applicationId` 和相关 import。
